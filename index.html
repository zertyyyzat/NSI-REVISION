🎯 Stratégie pour l'examen (1 heure)
⏱️
Lire tout le sujet en 5 minutes
Avant de coder quoi que ce soit, lis les 4 questions. Les Q1/Q2 sont souvent les plus simples (écrire une fonction de zéro). Les Q3/Q4 demandent de trouver et corriger un bug dans du code déjà écrit. Commence par les points faciles.
🧪
Toujours tester ton code
L'examinateur veut voir que ça tourne. Chaque sujet fournit des fonctions de test (test_xxx()). Lance-les ! Si tu as encore un bug, dis-le à voix haute à l'examinateur plutôt que de rester bloqué en silence — c'est prévu dans l'épreuve.
📝
Les questions "papier" (sans code)
Certains sujets ont une Q1 sur papier, par exemple "Est-ce que le RLE est forcément plus court ?" (Sujet 1). Ce n'est pas du code, c'est une réponse écrite avec un exemple. Réponds clairement et donne un contre-exemple si la réponse est "non".
🐛 Le bug le plus fréquent du bac NSI 2026
❌
Modifier une liste pendant qu'on la parcourt
C'est le piège numéro 1, présent dans les sujets 4, 10, 17... Imagine une liste [1, 2, 3, 4]. Si tu fais for x in liste: liste.remove(x), tu vas sauter des éléments parce que les index bougent. Fix : itérer sur une copie for x in liste[:]: ou construire une nouvelle liste avec une compréhension : [x for x in liste if condition]
❌
Oublier que tout est str quand on lit un CSV
Quand tu lis un fichier CSV, chaque valeur est une chaîne de caractères (str), même les nombres. Donc row['poids'] vaut "6.5" (str), pas 6.5 (float). Comparer "6.5" < 6.0 plante ! Fix : convertir au bon endroit : float(row['poids']) et int(row['id']).
❌
Ne pas gérer le cas "liste vide" → division par zéro
Une fonction de moyenne fait somme / len(liste). Si la liste est vide, len(liste) == 0 et tu as une ZeroDivisionError. L'énoncé dit souvent "retourner None si vide". Toujours commencer par : if not liste: return None
❌
Accéder au mauvais index dans un tuple
Dans le sujet 11, la fonction KNN retourne des tuples (distance, habitat). Le bug classique : écrire t['presence_renard'] au lieu de t[1]['presence_renard']. Un tuple s'accède par index numérique, pas par nom. Fix : vérifier l'ordre des éléments du tuple : t[0] = distance, t[1] = le dictionnaire.
🐍 Python : les 5 choses à connaître par cœur
1️⃣
Compréhension de liste — la super puissance Python
[expression for element in liste if condition]
Exemples : [e['salaire'] for e in employes if e['sexe']=='F'] → liste des salaires des femmes. [x*2 for x in [1,2,3]] → [2,4,6]. C'est la façon la plus claire et la plus lisible de filtrer/transformer une liste.
2️⃣
Les dictionnaires — lire et écrire
Lire : d['cle'] (plante si absent) ou d.get('cle', valeur_par_defaut) (sûr). Vérifier : 'cle' in d. Initialiser compteur : if k not in d: d[k] = 0. Parcourir : for cle, val in d.items(). Lister les clés : d.keys(), les valeurs : d.values().
3️⃣
La distance euclidienne (KNN)
from math import sqrt
sqrt((x1-x2)**2 + (y1-y2)**2) pour 2D.
Pour plus de dimensions, additionner tous les carrés. Important : ne mettre que les champs NUMÉRIQUES, pas "sexe" ou "nom" !
4️⃣
Conversion de types
int("42") → 42 | float("3.14") → 3.14 | str(123) → "123"
int("1010", 2) → 10 (binaire → décimal)
"5".isdigit() → True | ord('a') → 97 | chr(97) → 'a'
bin(97)[2:].zfill(8) → "01100001" (décimal → binaire 8 bits)
5️⃣
Les classes (POO)
__init__(self, ...) : appelé quand tu fais Classe(args), initialise les attributs avec self.attribut = valeur.
__str__(self) : appelé quand tu fais print(objet), doit retourner une str.
Toujours accéder aux attributs avec self. : self.nom, pas juste nom.
